# Boltzina Benchmark: Experimental Design

## Pipeline Overview

The boltzina benchmark evaluates **Boltz-2's neural affinity prediction** for virtual screening. The pipeline has three conceptual steps:

1. **Docking** — dock candidate ligands into the protein binding site to generate 3D poses
2. **Affinity scoring** — run each docked protein-ligand complex through Boltz-2's affinity module
3. **Evaluation** — compare predicted scores against known active/decoy labels

## Input Data

### Protein structure
- Source: **experimental crystal structure** from PDB (CIF format)
- Located at: `plate-vs/VLS_benchmark/zipped_uniprot_raw_cif/uniprot_{uid}/cif_files_raw/{pdb_id}.cif`
- The experimental CIF is processed into two forms:
  - **receptor.pdbqt** — for docking (Vina/Uni-Dock input)
  - **receptor.pdb** — for building protein-ligand complex CIF (Boltz-2 input)
- No structure prediction from sequence is needed — we use the known structure directly

### Ligands
- **Actives**: known binders from ChEMBL (filtered by similarity threshold 0.7, test split)
- **Decoys**: generated by DeepCoy (property-matched non-binders)
- Converted from SMILES → 3D PDB via RDKit ETKDG conformer generation

### Boltz-2 metadata artifacts
The Boltz-2 affinity module requires three metadata files that describe the protein-ligand system:

| Artifact | Content | How generated |
|----------|---------|---------------|
| `manifest.json` | Chain types, residue counts, ligand molecular weight, affinity target chain | Boltz's `process_input()` on a YAML describing protein sequence + reference ligand SMILES |
| `constraints/{uid}.npz` | RDKit bond/angle/chirality constraints for the reference ligand | Same `process_input()` call |
| `mols/{uid}.pkl` | Pickled RDKit Mol objects for ligand components (CCD lookup) | Same `process_input()` call |

These are **not** neural network outputs. They are deterministic parsing artifacts that `process_input()` generates from the input YAML in ~10 seconds on CPU. No GPU or structure prediction needed.

## Step 1: Docking

Dock each ligand against the protein receptor to generate a 3D binding pose.

### Methods available

| Method | Hardware | Speed (3000 ligands) | Pose quality |
|--------|----------|---------------------|-------------|
| AutoDock Vina | CPU (8 parallel) | ~2.4 hours | Higher (exhaustive search) |
| Uni-Dock (fast) | GPU | ~3 minutes | Lower (approximate) |
| Uni-Dock (detail) | GPU | ~15 minutes | Moderate |

### Output
- `out/{idx}/docked.pdbqt` — docked pose(s) with Vina energy scores

### Post-processing
Each docked pose is converted to a protein-ligand complex CIF for Boltz-2:
1. `obabel` — split multi-model PDBQT into individual PDB files
2. `pdb_chain` + `pdb_tidy` — assign chain ID (B) and clean formatting
3. `pdb_merge` — merge receptor PDB (chain A) with docked ligand PDB (chain B)
4. `maxit` — convert merged PDB → CIF → fixed CIF (`complex_fix.cif`)

## Step 2: Boltz-2 Affinity Scoring

### What the model receives
The `complex_fix.cif` file is parsed into a `StructureV2` object containing:
- Raw 3D atomic coordinates (protein + docked ligand)
- Bond connectivity
- Residue and chain metadata
- No pre-computed embeddings — everything is computed from scratch

### What the model does
Boltz-2 runs its **full neural network** on each complex:

```
complex_fix.cif
  → parse_mmcif() → StructureV2 (raw coords + structure)
  → Tokenize + Crop (max 256 tokens, 2048 atoms around binding site)
  → Input Embedder (coords + structure → initial embeddings)
  → Pairformer trunk (5 recycling iterations, full transformer)
  → Mask pairwise embeddings for protein-ligand interactions only
  → Compute distance features from input coordinates
  → Affinity-specific Pairformer stack
  → Affinity prediction heads
```

This is the same Boltz-2 trunk used for structure prediction, but:
- Structure module (diffusion) is **skipped** — input coordinates from docking are used directly
- Affinity heads replace the structure output heads

### What the model outputs

| Output | Type | Scale | Interpretation |
|--------|------|-------|----------------|
| `affinity_pred_value` | Continuous | ~0–2 (normalized) | Higher = stronger predicted binding. NOT on pChEMBL/IC50/Kd scale |
| `affinity_probability_binary` | Probability | 0–1 | P(binder). Learned binary classification, not calibrated |
| `docking_score` | Energy | kcal/mol | Vina/Uni-Dock physics-based score. Lower (more negative) = stronger binding |

**Important**: `affinity_pred_value` is an internal normalized value, not a physical affinity measurement. It cannot be directly compared to pChEMBL values or IC50.

## Step 3: Evaluation Metrics

### Classification metrics (active vs decoy ranking)

All three scores can be used to rank ligands. We evaluate how well each score separates actives from decoys:

| Metric | Definition | Score direction |
|--------|-----------|-----------------|
| **ROC-AUC** | Probability that a random active ranks above a random decoy | `affinity_pred_value`: higher = active. `docking_score`: lower (more negative) = active |
| **EF1%** | Enrichment factor at top 1%: (fraction of actives in top 1%) / (overall active rate) | Same as ROC-AUC |
| **EF5%** | Enrichment factor at top 5% | Same as ROC-AUC |

ROC-AUC works with any continuous score — it measures ranking quality, not calibration. A score of 0.5 = random, 1.0 = perfect separation.

### Regression metrics (affinity correlation, actives only)

For actives with known pChEMBL values:

| Metric | Definition |
|--------|-----------|
| **Spearman r** | Rank correlation between predicted score and pChEMBL value (actives only, minimum 5 data points) |

Note: Since `affinity_pred_value` is not on the pChEMBL scale, Pearson R² is not appropriate. Spearman rank correlation is the right metric — it measures whether the model ranks stronger binders above weaker ones regardless of scale.

### Scores to evaluate

Each of these is evaluated independently as a ranking metric:

1. **`docking_score`** (negated) — physics-based, from Vina/Uni-Dock
2. **`affinity_pred_value`** — Boltz-2 neural affinity
3. **`affinity_probability_binary`** — Boltz-2 binary binding probability
4. **Combined** (optional) — e.g., weighted sum or rank fusion of docking + affinity scores

## Experimental Conditions

### Factors to vary

| Factor | Options | Notes |
|--------|---------|-------|
| Docking method | Vina (CPU), Uni-Dock fast/balance/detail (GPU) | Affects pose quality and speed |
| Receptor structure for complex CIF | Experimental crystal, Boltz-predicted | Affects Boltz-2 scoring input geometry |

### Preliminary findings (Q07820, 75 ligands)

| Docking | Receptor for Boltz-2 | ROC-AUC (affinity) | ROC-AUC (docking score) |
|---------|---------------------|--------------------|------------------------|
| Vina | Predicted | 0.721 | 0.773 (negated) |
| Vina | Experimental | 0.290 | 0.773 |
| Uni-Dock fast | Experimental | 0.467 | — |
| Uni-Dock fast | Predicted | 0.372 | — |

Key observation: Boltz-2 affinity scoring is sensitive to the receptor structure format. The predicted structure produces higher ROC-AUC with Vina docking, suggesting the affinity module may have been trained on Boltz-predicted geometries.

## Directory Structure

```
results/
├── work_dirs/{uid}/
│   ├── {uid}.yaml                    # Boltz input (protein seq + reference ligand)
│   ├── vina_config.txt               # Docking box parameters
│   └── boltz_results_{uid}/
│       └── processed/                # Boltz-2 metadata artifacts
│           ├── manifest.json
│           ├── constraints/{uid}.npz
│           └── mols/{uid}.pkl
├── ligands/{uid}/
│   ├── actives/*.pdb                 # Known binders (3D conformers)
│   └── decoys/*.pdb                  # DeepCoy-generated non-binders
└── raw_results/{uid}/
    ├── receptor.pdbqt                # For docking
    ├── receptor.pdb                  # For complex building
    ├── config.json                   # Boltzina config
    ├── out/{idx}/
    │   ├── docked.pdbqt              # Docked pose(s)
    │   ├── docked_ligands/
    │   │   └── *_complex_fix.cif     # Protein-ligand complex for Boltz-2
    │   └── done                      # Completion flag
    ├── boltz_out/predictions/        # Boltz-2 scoring outputs
    │   └── {record_id}/
    │       ├── pre_affinity_*.npz    # Parsed complex (input to model)
    │       └── affinity_*.json       # Predicted affinity (model output)
    └── boltzina_results.csv          # Final per-ligand scores
```
